Before starting the game:
1.  Run the server executable.
    It will tell you the IP address students can connect to LATER.
2.  Run the Electron application.
    It should load the game with an empty map (no colors).
3.  dev -> LOADMAP then select the right map from Steven/games/docs/conquest/maps.
4.  SERVER -> BUCKETS to set the buckets file (it is in Steven/secrets)
5.  SERVER -> MAXATTACKS -> 1 to start with.
    Can increase later.

Students can now start joining to the IP address.
Ask them to close all apps and tabs.
Connect to "Student" wifi, disconnect 5G.
1.  If a student has an (X) next to their name then they lost connection.
    Small handheld devices may lose connection repeatedly. It is not an issue in general.
    The students might be annoyed (the page will refresh 
    instead of submitting their answers so they will need to type it in again).
2.  SERVER -> START to begin the game.

Student management:
Click on team-colored banner, then pick a student.

During the game:
SERVER -> INVALIDATE -> QUESTIONID to remove a faulty question from the question bank.
Gradually increase SERVER -> MAXATTACKS throughout the game. Use your judgement.

Ending the game:
1.  SERVER -> END -> Yes to end the game.
2.  SERVER -> DISPLAY to show students the server screen.
3.  SERVER -> DOWNLOAD to let students download a copy of the server screen.

After the game:
1.  Close both the Electron and the server app.
2.  Send me a copy of:
- Steven/clipped/conquestEnd(numbers).json
- Steven/clipped/diagnostic(numbers).json
- Steven/games/docs/record.txt
- You can make a timelapse out of the "timelapse" pictures in the Steven/clipped folder 
  if you wish on https://sisik.eu/images-to-video



Client (student)-side issues & fixes:

Most common problems: too many open tabs, too many open apps, background apps using up the bandwidth.
Solution to each of these: turn off 5G, close all applications and close all tabs.

Can't enter fullscreen - tap the student's name in the right-upper corner.
Image won't center, is too zoomed in, is too zoomed out - tap the gray header "Conquest game" several times.

Stuck on loading screen - tap the text several times to flush and refresh.

Page won't load entirely: double-check the IP address, and make sure that autocomplete did not add any ports
(it tends to correct 10.40.1.219 to the incorrect 10.40.1.2019:8000 for example).

Slow to load or has noticeable input lag - throttling due to low battery or too many other apps running.

Anything else: add #d at the end of the URL to allow "debug" info to be printed on the page,
then call me for help or take pictures of the errors log.

Known issues (for later reference)

Windows:
admin privileges must be granted to node to be stable
iis might be using port 80. Kill it with:
iisreset /stop
OR: net stop was /y OR: net stop w3svc /y
otherwise: netstat -ano | findstr :80 and kill whatever's causing trouble

Mac:
sudo lsof -i :80
If it's httpd / Apache:
sudo apachectl stop
(full prevent) sudo launchctl unload -w /System/Library/LaunchDaemons/org.apache.httpd.plist
sudo kill <PID>
sudo kill -9 <PID>
sudo nice -n -20 ~/Documents/Steven/your_app_name
